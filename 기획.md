# 프로젝트 정의서 / Project Charter

PMI / PMBOK 양식 기준에 참고하여 아래와 같이 프로젝트 헌장을 제안합니다.

## 1. 프로젝트 개요
- 프로젝트명: "골목대장"
- 프로젝트 목적 : 머신러닝과 AI 에이전트를 활용하여 창업 지식이 없는 초기 자영업자가 이용할 수 있는 프랜차이즈 창업 플랫폼을 만든다.
- 프로젝트 배경 
    - 취업대신 창업을 선택하는 20대, 불황으로 은퇴/정리해고 후 창업을 선택하는 50대 등은 창업에 대한 정보를 얻기 부족하다. 
    - 이와 같은 문제를 해결하기 위해 창업에 대한 정확한 예측과 정보를 제공하되, 편의성보다 전문성에 중점을 두어 사용자 스스로 구체적인 입지 선정 및 업종 선정을 할 수 있도록 한다. 
    - 초기 창업자는 프랜차이즈를 통해 창업하는 것이 성공률이 높다. 프랜차이즈와 연동하여 프랜차이즈-창업주를 연결하는 플랫폼으로 성장시킨다.

## 2. 프로젝트 목표
- 정량적 목표
    - 사용자 선호도를 통한 창업 입지 추천 머신러닝 모델을 1개 이상 서비스에 적용한다.
    - 프랜차이즈 창업의 예측 모델(단기 또는 중/장기)을 1개 이상 서비스에 적용한다.
    - AI 에이전트 하네스를 개발하여 지도 및 사용자 선호도와 연동하고 RAG를 구축한다.
- 정성적 목표
    - 프랜차이즈 창업자가 입지 추천, 업종 추천으로 이어지는 사용자 플로우를 구축한다.
    - 프랜차이즈 창업자가 AI 에이전트와 함께 구체적인 창업 계획을 세우고 관리하는 사용자 플로우를 구축한다.
- 성공 기준
    - 유지보수 및 부분 재학습 가능한 ML 예측 모델 1개 이상
    - 사용자의 관심사를 관리하는 AI 에이전트 및 RAG 연동 1개 이상


## 3. 상위 수준 요구사항
- 핵심 사용자 요구
    - 사용자는 자신이 원하는 창업 조건을 입력하여 입지와 프랜차이즈를 추천받을 수 있어야 한다.
    - 사용자는 AI 에이전트의 도움을 받아 창업 리포트 작성 및 일정관리를 할 수 있어야 한다.
- 핵심 서비스 요구
    - 상권은 행정동 단위로 데이터를 수집하고 Java 21 버전을 통해서 관리되어야 한다. 
    - AI 에이전트 기능은 RAG를 통해 최신 정보를 전달할 수 있어야 한다.
- 핵심 기술 요구
    - AI 에이전트는 객관적으로 평가 가능한 벤치마크를 통해 개선할 수 있어야 한다.
    - 머신러닝 모델은 FastAPI를 통해 배포되고 재학습 가능해야 한다.


## 4. 프로젝트 범위

### 4.1 프로젝트 설명
본 프로젝트는 서울 행정동 단위 상권 데이터와 프랜차이즈/업종 데이터를 기반으로, 초기 예비 창업자가 자신의 조건에 맞는 창업 입지와 프랜차이즈를 탐색할 수 있도록 지원하는 AI 기반 창업 의사결정 플랫폼을 개발한다.

사용자는 창업 조건을 입력하고, 서비스는 이를 바탕으로 적합한 입지와 업종/프랜차이즈를 추천한다. 또한 AI 에이전트를 통해 창업 관련 질의응답과 리포트 생성을 지원한다.

### 4.2 포함 범위
- 서울 행정동 단위 상권 데이터 조회 및 분석
- 사용자 창업 조건 입력 및 온보딩
- 사용자 조건 기반 입지·업종/프랜차이즈 추천
- 지도 기반 상권 탐색 및 상권 상세 정보 제공
- AI 에이전트 기반 질의응답 및 리포트 생성
- 사용자별 리포트 및 관심 조건 저장
- 머신러닝 기반 추천 또는 예측 모델 연동

### 4.3 제외 범위
- 서울 외 지역 상권 분석
- 실제 창업 계약, 가맹 계약, 결제, 중개 기능
- 프랜차이즈 본사와의 실시간 데이터 연동
- 대출, 보험, 세무, 법무 등 전문 자문 서비스
- 상용 서비스 수준의 추천 정확도 보장
- 모바일 네이티브 앱 개발

### 4.4 주요 산출물
- AI 기반 창업 입지·프랜차이즈 추천 웹 서비스
- 서울 행정동 기반 상권 데이터베이스
- 추천/예측 머신러닝 모델
- RAG 기반 리포트 생성 기능
- AI 에이전트 자체 벤치마크 및 측정치
- 지도 기반 상권 탐색 화면
- 프로젝트 문서 및 발표 자료

## 5. 가정 및 제약사항
### 5.1 가정사항
- 창업 업종과 프랜차이즈는 높은 상관도를 지닌다.
- 프랜차이즈의 각 브랜드는 서로 구별되는 정성/정량적 차이를 지니며, 이를 사용자 선호도와 함께 평가할 수 있다.
- 사용자와 AI의 대화 내용을 서비스 내에서 활용하는 것은 공정활용에 해당한다.
- 상권과 관련된 데이터가 충분하고, 이를 최신 데이터와 연동하여 트렌드 예측이 가능하다.


### 5.2 제약사항
- 서비스 범위는 서울로 한정한다.
- 프랜차이즈와의 계약이나 법률/세무 서비스는 직접 지원하지 않는다.
- AI 에이전트를 위한 법률/행정/규칙 RAG는 구축하지 않고 웹 검색으로 대체한다.

### 5.3 의존성
- 서울시 상권분석서비스 및 서울 열린데이터광장 공공데이터에 의존한다.
- 공정거래위원회 및 공공데이터포털의 프랜차이즈 관련 공개 데이터에 의존한다.
- AI 에이전트는 파이썬 Langgraph의 자체 API 규칙에 의존한다.
- 지도 기반 탐색 기능은 행정동 공간 데이터와 지도 API 또는 지도 시각화 라이브러리에 의존한다.

## 6. 주요 리스크

### 6.1 예상 리스크
- 공공데이터의 최신성·결측치·정합성 문제로 인해 추천 및 예측 모델의 품질이 낮아질 수 있다.
- 프랜차이즈 공개 데이터가 부족하여 브랜드별 차이와 추천 근거를 충분히 설명하지 못할 수 있다.
- 머신러닝 모델이 실제 창업 성과를 충분히 설명하지 못하거나, 학습 데이터에 과적합될 수 있다.

### 6.2 리스크 영향도
- 데이터 품질 문제는 입지 추천과 트렌드 예측의 신뢰도를 낮추므로 영향도가 높다.
- 프랜차이즈 데이터 부족은 프로젝트의 핵심 차별점 약화로 이어질 수 있으므로 영향도가 높다.
- 머신러닝 모델의 낮은 설명력과 과적합은 추천 결과의 신뢰도와 캡스톤 평가에 영향을 줄 수 있으므로 영향도가 높다.

### 6.3 대응 방안
- 데이터 출처, 수집 시점, 결측치 처리 기준, 전처리 과정을 문서화한다.
- 프랜차이즈 데이터가 부족한 경우 업종 단위 데이터를 보조 지표로 활용한다.
- 머신러닝 모델은 정확도뿐 아니라 설명 가능성, 검증 데이터 성능, 기준 모델 대비 개선 여부를 함께 평가한다.
